Ledger

Value created vs. value captured, layer by layer, for a value chain. Where capture exceeds creation, the layer is extracting rent. Where creation exceeds capture, value is leaking to a layer that isn't the one doing the work — usually because it lacks a real moat. Every layer cites the mechanism, the capture metric (or an honest "asserted" flag if none is disclosed), and a moat from the 7 Powers taxonomy or "none".

Update Protocol

This tool is designed to be maintained by AI sessions. The update protocol is in UPDATE_PROTOCOL.md. In brief:

1. Never edit engine code for data updates. All data lives in /data/*.json. The engine reads it; it never owns it.

2. Every claim touched must be re-verified, not assumed. Downgrade cf to "md" rather than guess.

3. After any data edit: run node js/validate.js, bump meta.json version, set last_updated to today.

4. The Data Health panel (⊙ in the header) shows which nodes are past their review window — that list is the update queue for the next session.

5. A weekly CI job regenerates that same queue for free, with no AI and no API key — see PATROL.md. Turning it into actual edits happens on demand: paste this repo to a chat AI and point it at AGENT_UPDATE.md.

Survivorship Bias Rules

dead_bodies is mandatory for every Tier C/P entry. If you add a winning bet without listing who made the same bet and died, the validator fails.

Every future_paths entry requires a falsifier — an observable that would kill the path. Probabilities without falsifiers are not forecasts; they are opinions.
